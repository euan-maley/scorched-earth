## Key Decisions

- **Sleep-aware windows.** `WINDOWS LEFT` / `REC. BURN` / the green-amber-off call all counted
  windows you sleep through. Now usable windows = raw × (active hours / 24), and active hours are
  LEARNED from when the statusline fires (16h fallback). Applied in `core.compute` so statusline,
  CLI, and sitrep stay consistent.
- **R guardrails.** A noisy R=0.25 (from 2 readings) was flipping the verdict to OFF when the user
  should be GREEN. Fixed in `calibrate.py`: discard out-of-band per-pair estimates (1–20%) and hold
  the safe default until ≥3 clean pairs. R can't be read live (API gives only %, not caps), so
  guard-railed estimation is the right model.
- **Forecast capacity cap.** `habits.forecast` now caps `expected_remaining` at
  `min(weekly_left, max_burnable)` so projected leftover is never understated.
- **Live recompute in the report.** `report._stats` recomputes the recommendation AND forecast from
  the snapshot (not the cached `state.json` values), so verdict/reason/projections always reflect
  current math and voice. Canonical `HEADLINE` moved to `core.py`, shared by CLI + sitrep.
