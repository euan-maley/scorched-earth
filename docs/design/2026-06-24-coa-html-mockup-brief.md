# Design Brief: Course of Action (COA) HTML mockup

**For:** Claude design (HTML mockup generation)
**Date:** 2026-06-24
**Context:** Scorched Earth, a Claude-usage tool. This brief is for the **COA report**, the visual presentation of a budget-matched work plan. It is the sibling of the existing **sitrep** and must feel like the same product.

## What this is

Scorched Earth signals when to burn down your weekly Claude budget. The COA advisor takes that signal and produces a **Course of Action**: a ranked list of expensive jobs in your repos, sized to the budget you have left, that you can run now or queue. The COA report is the HTML presentation of one such plan.

Right now the renderer (`src/scorched_earth/coa_report.py:render_html`) emits a plain dark table. We want it to look like a military operations briefing rendered in the same 8-bit scorched-earth war-HUD style as the sitrep. This brief asks you to design that mockup.

## Visual language (match the sitrep)

The whole product is an 8-bit war / scorched-earth aesthetic. Reuse it.

- **Palette:** background `#0b0705`; fire accents `#ff3b1f`, `#ff6a1f`, `#ff8a1f`, `#ffd24a`, `#ffe7a0`; muted text `#f4e4c8` / `#e9c08a`; HUD teal `#86abab` / `#6f8a8a`; charred `#1a120b`; soil/brown `#6b4a2b`.
- **Type:** monospace, pixel feel (`ui-monospace, SFMono-Regular, Menlo, Consolas, monospace`), wide letter-spacing on headings.
- **Frame:** corner HUD brackets at the page corners, a faint CRT scanline overlay and a vignette, like the sitrep panel.
- **Headline treatment:** the title uses the animated fire-gradient sweep (the same `.firetext` effect as "BURN IT ALL").
- **Register:** military. The feature names are **COA = Course of Action** and **ROE = Rules of Engagement**. Voice is a war-general giving orders. Existing verdict line example: "Torch it all. Leave them nothing."

Two existing assets to draw from (same repo / same look):
- The sitrep HTML, generated by `src/scorched_earth/report.py` (the pixel-farm field, the HUD stat cards, the scanline/vignette/corner-bracket frame, and the burn-mode fire).
- The standalone fire engine (`~/Downloads/scorched-earth-burn-fire.html`): the Doom-fire canvas plus rising embers and smog. Reuse it for accents if it helps (e.g. a thin ember line under the header).

## What the report must present

A COA is a budget envelope plus a ranked queue of jobs that fills it, plus the jobs that spilled over.

1. **Header.** "COURSE OF ACTION", a sector/date stamp (e.g. `// SECTOR 07 · 2026-06-24`), the corner-bracket frame, fire-gradient title.

2. **The budget envelope (the "ammunition").** The single most important number: how much burn is available and how much this plan commits. Think an ammo belt / fuel gauge: `spent` of `available` windows committed, with the unfilled remainder shown as still-loaded. If the plan leaves budget unspent, echo the product's core anxiety: unspent budget is **forfeit at reset**. Tie the accent color to the verdict (GREEN burn-it-all = hottest).

3. **The queue (the "battle plan / target list").** The ranked, ordered jobs, in run order. Each job is one objective and shows:
   - a rank number (1, 2, 3 …),
   - the **title**,
   - a **tier badge** (S / M / L / XL) where XL reads hottest/biggest and S coolest/smallest,
   - the **type** (test / docs / refactor / perf / audit / …) as a small tag,
   - the **cost** (`estWindows`, in window-units) and the **value**,
   - the **rationale** (one line on why it is worth running),
   - the **launch** command (monospace, copyable feel, the thing the user runs).
   Treat these like mission cards or a stacked target list. Tier should drive visual weight: an XL "overnight" job should feel like a major operation, an S job like a quick strike.

4. **Left on the table (the spillover / "forfeit").** Jobs that did not fit the budget or were excluded by the rules of engagement. Render them charred / dimmed, like scorched ground. This is the cost of not burning more.

5. **The verdict line.** The COA `note` string (e.g. "Queued 3 job(s), ~2.0 of 2.5 windows."), styled as the commander's summary.

Optional flourish if it fits the aesthetic without clutter: a small ROE chip showing the active confines (e.g. "ROE: max 2 windows · no main · tests after"), so the viewer sees what bounded the plan.

## Technical constraints (important)

- **Single self-contained HTML file.** No external assets, fonts, CDNs, or network calls. All CSS/SVG/JS inline. It must open offline and ship inside the plugin.
- **It is a template, not a one-off.** Python computes the data and injects it; JS renders. Follow the sitrep's exact pattern: a single JSON blob assigned to a `const COA = {…}` near the bottom, which the page's JS reads to build the DOM. Python will replace that blob (or specific `__TOKENS__`) at render time. Keep all data access going through that one object so the Python side has a clean seam.
- **Determinism / no clock:** the date is passed in as a preformatted string (`generatedAt`); do not call `Date()` for the stamp. Live countdowns are not needed here (unlike the sitrep).
- **Respect `prefers-reduced-motion`:** any fire/ember animation must freeze under it.
- **Responsive enough** to read at ~640 to 1100px wide.

## Data contract

Design the JS against exactly this shape (Python fills it). All numbers are window-units except `value`.

```js
const COA = {
  generatedAt: "2026-06-24",
  verdict: "GREEN",                 // GREEN | AMBER | OFF | UNKNOWN  -> drives accent
  note: "Queued 3 job(s), ~2.0 of 2.5 windows.",
  envelope: { available: 2.5, spent: 2.0 },
  roe: { maxWindows: 2.5, allowedTypes: null, goals: ["harden error handling"] },  // may be partial
  queue: [
    { id: "cov-core", rank: 1, title: "Exhaustive test coverage for core/", type: "test",
      tier: "L", estWindows: 1.5, value: 9,
      rationale: "core/ has 38% coverage and gates every other module.",
      launch: "claude -p 'Raise core/ test coverage to 90%, TDD, one module at a time.'",
      status: "proposed" },
    { id: "err-audit", rank: 2, title: "Error-handling audit across the I/O layer", type: "audit",
      tier: "M", estWindows: 1.0, value: 7,
      rationale: "Several bare excepts swallow failures silently.",
      launch: "claude -p 'Audit and harden error handling in the I/O layer.'",
      status: "proposed" }
  ],
  skipped: [
    { id: "rewrite-render", title: "Rewrite the render pipeline", type: "refactor",
      tier: "XL", estWindows: 4.0, value: 6,
      rationale: "Big payoff but does not fit tonight's budget.",
      reason: "over budget" }
  ]
};
```

Notes on fields:
- `verdict` maps to the header accent. GREEN is the hottest (full burn), AMBER warm, OFF/UNKNOWN cooler.
- `tier` (S/M/L/XL) is the human label; `estWindows` is the real cost. Use `tier` for the badge, `estWindows` for the gauge math.
- `skipped[].reason` is a short string ("over budget", "disallowed type", "over per-job cap").
- `roe` may have null/missing keys; render only what is present.
